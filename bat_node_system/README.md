# Bat Node System v2 Starter CodeThis package contains starter code for the bat acoustic monitoring architecture:* `server/` FastAPI + SQLite secure ingest server* `esp32/` ESP32 secure check-in firmware skeleton* `audiomoth/` AudioMoth custom firmware pseudocode for UART + SD arbitrationThe design intentionally avoids node-side full-file hashing. The server verifies byte counts, parses WAV files, computes the archive SHA-256, optionally creates FLAC files, and then issues signed delete authorization.\---## 1\. Server setup on WindowsOpen PowerShell in the `server` folder.```powershellpy -3 -m venv .venv. .\\\\\\\\.venv\\\\\\\\Scripts\\\\\\\\Activate.ps1pip install -r requirements.txt```Create the first node credential:```powershellpython manage\\\\\\\_node.py create BATNODE\\\\\\\_001 "Bench Node 1" --lat 46.8772 --lon -96.7898 --location-label "Bench"```Copy the printed values into the ESP32 sketch:```textNODE\\\\\\\_ID=...KEY\\\\\\\_ID=...DEVICE\\\\\\\_SECRET=...```Run the server:```powershelluvicorn bat\\\\\\\_server:app --host 0.0.0.0 --port 8000```Open:```texthttp://127.0.0.1:8000/dashboard```Default dashboard login:```textadmin / change-me-now```Change it before any remote exposure.You can also run:```powershell.\\\\\\\\run\\\\\\\_server\\\\\\\_powershell.ps1```\---## 2\. ESP32 setupInstall in Arduino IDE:* ESP32 board support* ArduinoJson libraryOpen:```textesp32/bat\\\\\\\_node\\\\\\\_phase1\\\\\\\_secure\\\\\\\_checkin.ino```Set:```cpp#define WIFI\\\\\\\_SSID      "..."#define WIFI\\\\\\\_PASSWORD  "..."#define BASE\\\\\\\_URL       "http://YOUR\\\\\\\_PC\\\\\\\_LAN\\\\\\\_IP:8000"#define NODE\\\\\\\_ID        "BATNODE\\\\\\\_001"#define KEY\\\\\\\_ID         "key-1"#define DEVICE\\\\\\\_SECRET  "..."```Flash the ESP32. Check Serial Monitor at 115200 baud.For LAN bench testing, HTTP is acceptable. For field deployment, use HTTPS through a tunnel or cloud relay.\---## 5\. AudioMoth firmwareThe file:```textaudiomoth/audiomoth\\\\\\\_uart\\\\\\\_firmware\\\\\\\_pseudocode.c```is not drop-in firmware. It defines the behavior that custom AudioMoth firmware must implement:* `GET\\\\\\\_STATE`* `LOCK\\\\\\\_REQ`* `RELEASE\\\\\\\_LOCK`* `GET\\\\\\\_TIME`* `SET\\\\\\\_TIME`* `LIST\\\\\\\_FILES`* `READ\\\\\\\_CHUNK`* `DELETE\\\\\\\_FILE`The important rule is fixed:```textRecording and SD writes have priority over ESP32 upload/delete requests.```\---## 6\. Current endpoint summaryPublic:```textGET /healthGET /v1/public/server\\\\\\\_time```Device, HMAC-signed:```textPOST /v1/device/heartbeatGET  /v1/device/{node\\\\\\\_id}/commandsPOST /v1/device/{node\\\\\\\_id}/commands/{command\\\\\\\_id}/ackPOST /v1/device/time\\\\\\\_checkPOST /v1/files/manifestPOST /v1/uploads/initPUT  /v1/uploads/{upload\\\\\\\_id}/chunks/{chunk\\\\\\\_index}GET  /v1/uploads/{upload\\\\\\\_id}/statusPOST /v1/uploads/{upload\\\\\\\_id}/completeGET  /v1/nodes/{node\\\\\\\_id}/delete\\\\\\\_authorization?manifest\\\\\\\_id=...POST /v1/nodes/{node\\\\\\\_id}/delete\\\\\\\_confirm```Dashboard/admin, HTTP Basic:```textGET  /dashboardPOST /admin/commands/{node\\\\\\\_id}/{command\\\\\\\_type}```\---## 7\. Delete policyThe server authorizes deletion only when the file has at least:* complete byte count* parseable WAV* server-side SHA-256* committed database rowOptional stricter gates:```textREQUIRE\\\\\\\_FLAC\\\\\\\_BEFORE\\\\\\\_DELETE=1REQUIRE\\\\\\\_BACKUP\\\\\\\_BEFORE\\\\\\\_DELETE=1```Set those after FLAC and backup are tested.\---## 8\. No open inbound home portsFor bench testing, run locally on your LAN.For deployment, do not port-forward your home PC. Use one of these patterns:1. Cloud HTTPS ingest API + home PC outbound sync worker.2. Cloudflare Tunnel or similar outbound tunnel from home PC.3. VPN overlay for admin only.The ESP32 should talk to a stable HTTPS endpoint. The home PC should connect outbound.\---## 9\. Next code phaseAfter phase-1 heartbeat is stable, implement these in order:1. ESP32 AudioMoth UART binary framing.2. AudioMoth custom firmware `LIST\\\\\\\_FILES` and `READ\\\\\\\_CHUNK`.3. ESP32 manifest upload from AudioMoth file list.4. ESP32 resumable upload chunks.5. Server-side weather worker.6. Signed delete authorization consumption on ESP32.7. AudioMoth `DELETE\\\\\\\_FILE` implementation.\---# User-friendly dashboard appA separate Streamlit dashboard is included in:```textbat\\\\\\\_node\\\\\\\_system/dashboard/bat\\\\\\\_dashboard\\\\\\\_app.py```It provides app-like access to the SQLite database without forcing you to inspect tables manually.## Windows quick start```powershellcd dashboard.\\\\\\\\run\\\\\\\_dashboard\\\\\\\_powershell.ps1```or double-click:```textdashboard/run\\\\\\\_dashboard\\\\\\\_windows.bat```Default database path:```textserver/bat\\\\\\\_nodes\\\\\\\_v2.db```The dashboard includes:* Node overview* Battery/status cards* Satellite map using saved node coordinates* File browser* SD cleanup audit* Telemetry charts* Command queue buttons* Diagnostics* Raw read-only database viewerDo not expose the Streamlit dashboard directly to the public internet. Use it locally, over Tailscale/ZeroTier, or behind Cloudflare Access.
